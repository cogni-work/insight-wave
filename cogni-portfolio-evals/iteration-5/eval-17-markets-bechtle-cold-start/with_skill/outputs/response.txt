## Prerequisite Gap: No Features or Products Defined

Before we can define target markets with any rigour, we need to know what Bechtle is actually selling through this portfolio. Right now the `products/` and `features/` directories are empty -- there are no products, no features, and no capability taxonomy mappings. That matters because market definition should start from "who has the problem our features solve?", not from "what industry verticals exist?" Without features, we'd be drawing segments on a whiteboard with nothing to put inside the propositions.

**This is a blocking prerequisite.** I will not generate market JSON files because every market I'd create today would be an academic exercise -- it would have zero features to map into propositions, zero basis for sizing a SAM (serviceable to whom, for what?), and zero differentiation story. The moment you add features, the market definitions would need to be reworked anyway.

## What I'd Recommend Instead

Define products and features first. Bechtle AG is a EUR 6B+ German IT services provider with ~15,000 employees, operating across managed infrastructure, cloud, digital workplace, security, and IT e-commerce. That's a broad portfolio. The right sequence is:

1. **Run the `portfolio-scan` or `features` skill** to populate products and features -- either from existing collateral (upload Bechtle capability decks, service descriptions, or website content to `research/`) or by manually defining them.
2. **Then come back to markets.** Once I can see what Bechtle's portfolio actually contains, I can ground market recommendations in real feature-market fit rather than generic industry segmentation.
