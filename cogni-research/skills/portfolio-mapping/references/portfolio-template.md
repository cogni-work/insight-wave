## Service Horizons Legend

| Horizon | Timeframe | Characteristics |
|---------|-----------|-----------------|
| **Current** | 0-1 years | Generally available, proven deployments, established pricing |
| **Emerging** | 1-3 years | Pilot/beta, limited availability, early adopter pricing |
| **Future** | 3+ years | Announced, conceptual, R&D phase, no fixed pricing |

## Discovery Status Legend

| Status | Meaning |
|--------|---------|
| **Confirmed** | Provider offers this service (evidence found) |
| **Not Offered** | No evidence found for this category |
| **Emerging** | Announced or pilot status (not yet GA) |
| **Extended** | Provider-specific variant beyond standard taxonomy |

## Table Column Reference

| Column | Content |
|--------|---------|
| Name | Product/service name as marketed |
| Description | 1-2 sentence summary of what it does |
| Domain | Source domain where service was found (e.g., t-systems.com) |
| Link | Direct URL to source page in markdown format: `[Link](url)` |
| USP | Unique selling proposition / differentiators |
| Provider Unit | Business unit offering this (e.g., T-Systems, MMS) |
| Pricing Model | subscription, usage-based, project-based |
| Delivery Model | Onshore, nearshore, offshore, hybrid |
| Partners | Key technology partnerships |
| Verticals | Target industries |
| Horizon | Current, Emerging, or Future |
