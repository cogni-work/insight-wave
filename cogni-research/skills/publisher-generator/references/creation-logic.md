# Publisher Creation Logic

## Overview

Extract publisher metadata from source entities and create structured publisher entities with type detection, organization classification, and automatic deduplication.

## Input Requirements

**Source Entities (`07-sources/data/`):**

- Valid YAML frontmatter required
- Key fields:
  - `id`: Source entity ID (e.g., "source-climate-bonds-report-abc123") - **Required for linking**
  - `domain`: Publisher domain (e.g., "nytimes.com") - **Required**
  - `authors`: Comma-separated author names (optional)
  - `url`: Full source URL (canonical field name per entity-templates.md)
  - `publisher_id`: Wikilink to expected publisher (generated by source-creator using generate-publisher-id.sh)

**Example Source Entity:**
```yaml
---
id: source-climate-bonds-report-abc123
name: "Green Bond Market Report 2024"
title: "Green Bond Market Report 2024"
url: "https://www.climatebonds.net/resources/reports/2024"
domain: "climatebonds.net"
access_date: "2024-01-15"
authors: Dr. Jane Smith, John Doe
reliability_tier: null
tags: [source, source-type/academic]
dc:creator: source-creator
dc:title: "Green Bond Market Report 2024"
dc:source: "https://www.climatebonds.net/resources/reports/2024"
dc:type: source
---
```

## Type Detection Logic

### Individual Publishers

**Triggers:**
- `authors` field present and non-empty
- Author names not generic ("Staff", "Editorial Board", "Anonymous")
- Multiple authors split by comma

**Extraction Process:**
```bash
# From source entity
authors: "Dr. Jane Smith, John Doe"

# Split and normalize
→ ["Dr. Jane Smith", "John Doe"]

# Create separate entities
→ publisher-dr-jane-smith-{hash}.md
→ publisher-john-doe-{hash}.md
```

**Entity Structure:**
```yaml
---
entity_type: publisher
publisher_type: individual
name: Dr. Jane Smith
enriched: false
created_date: 2025-11-07T09:00:00Z
source_references:
  - source-climate-bonds-report-abc123
tags: [publisher, publisher-type/individual]
dc:creator: publisher-generator
dc:type: publisher
---

## Publisher: Dr. Jane Smith

**Type**: Individual

**Affiliations**: Not yet documented

### Sources by This Publisher
- [[cogni-research/climate-finance-2024/07-sources/data/source-climate-bonds-report-abc123]]
```

**Note:** Wikilinks are generated using `generate-wikilink.sh` utility for workspace-relative paths. See "Wikilink Generation for Sources" section below for implementation details.

### Organization Publishers

**Triggers:**
- NO `authors` field OR empty/generic authors
- Domain-based attribution present
- Organizational signals in source metadata

**Extraction Process:**
```bash
# From source entity
domain: "climatebonds.net"

# Extract organization name from domain
→ "Climate Bonds Initiative"

# Classify organization type
→ organization_type: "ngo"

# Create entity
→ publisher-climate-bonds-initiative-{hash}.md
```

**Organization Type Classification:**

| Type | Domain Indicators | Examples |
|------|-------------------|----------|
| `multilateral_development_bank` | worldbank, imf, adb, ebrd | worldbank.org |
| `government_agency` | .gov, .eu, treasury, sec, oecd | treasury.gov |
| `ngo` | oxfam, amnesty, greenpeace, wwf | oxfam.org |
| `industry_association` | iso, ieee, icma | ieee.org |
| `academic_institution` | .edu, university, college | mit.edu |
| `financial_institution` | bank, jpmorgan, goldman | jpmorgan.com |
| `news_organization` | nytimes, reuters, bloomberg | nytimes.com |
| `international_organization` | un, who, wto, nato | un.org |
| `think_tank` | brookings, rand, chatham | brookings.edu |
| `private_company` | (default) | microsoft.com |

**Classification Logic:**
```python
def classify_organization(domain, name):
    # Check domain patterns
    if matches(domain, ['worldbank', 'imf', 'adb']):
        return 'multilateral_development_bank'

    if domain.endswith('.gov') or matches(domain, ['treasury', 'sec']):
        return 'government_agency'

    if matches(domain, ['oxfam', 'amnesty', 'greenpeace']):
        return 'ngo'

    # ... (check all patterns)

    # Default
    return 'private_company'
```

## Deduplication

**Mechanism:** Semantic slug with hash for uniqueness + glob pattern for deduplication

**Process:**

1. Generate semantic slug from publisher name
   ```bash
   name="Dr. Jane Smith"
   slug=$(echo "$name" | tr '[:upper:]' '[:lower:]' | tr ' ' '-' | sed 's/[^a-z0-9-]//g')
   → slug="dr-jane-smith"
   ```

2. Generate 8-character hash from name
   ```bash
   hash=$(echo "$name" | md5sum | cut -c1-8)
   → hash="a1b2c3d4"
   ```

3. Create deterministic ID combining slug and hash
   ```bash
   publisher_id="publisher-$slug-$hash"
   → "publisher-dr-jane-smith-a1b2c3d4"
   ```

4. Check if entity exists using glob pattern
   ```bash
   existing=$(ls 08-publishers/data/publisher-$slug-*.md 2>/dev/null | head -1)
   ```

5. If exists → Reuse
   ```bash
   # Read existing entity
   # Parse source_references array
   # Append source_id if not already present using Edit tool
   # Update last_modified date
   # Increment reused counter
   ```

6. If new → Create via create-entity.sh
   ```bash
   # ⛔ NEVER use Write tool - use create-entity.sh
   # ⛔ MUST pass --entity-id from generate-publisher-id.sh
   "${CLAUDE_PLUGIN_ROOT}/scripts/create-entity.sh" \
     --project-path "$PROJECT_PATH" \
     --entity-type "08-publishers" \
     --entity-id "$PUBLISHER_ID" \
     --data "$PUBLISHER_JSON" \
     --json
   # Increment created counter
   ```

## ⛔ CRITICAL: source_references Must Be Populated

**Every publisher MUST have at least one source in `source_references` array.** Empty `source_references: []` is a bug.

When creating a publisher:
1. Extract `id` field from source file YAML frontmatter
2. Include source ID in `source_references` array in entity JSON
3. Include wikilink to source in Related Sources section

**Correct entity JSON:**
```json
{
  "frontmatter": {
    "source_references": ["source-climate-bonds-report-abc123"],
    ...
  },
  "content": "## Publisher...\n\n### Related Sources\n- [[07-sources/data/source-climate-bonds-report-abc123]]"
}
```

**WRONG (will break linkage):**
```json
{
  "frontmatter": {
    "source_references": [],
    ...
  },
  "content": "## Publisher...\n\n### Related Sources\n"
}
```

**Statistics Tracking:**
- `publishers_created`: New entities created
- `publishers_reused`: Existing entities reused
- `publishers_failed`: Extraction/creation failures

## Name Extraction

### From Authors Field

**Patterns:**
```bash
# Simple names
"Jane Smith" → "Jane Smith"

# With titles
"Dr. Jane Smith" → "Dr. Jane Smith"

# With affiliations (remove)
"Jane Smith, MIT" → "Jane Smith"

# Multiple authors (split)
"Jane Smith, John Doe" → ["Jane Smith", "John Doe"]
```

**Normalization:**
- Trim whitespace
- Remove email addresses
- Remove affiliations in parentheses
- Preserve titles (Dr., Prof., etc.)

### From Domain

**⚠️ CRITICAL:** Use `generate-publisher-id.sh` for organization name extraction to ensure ID consistency with source-creator.sh:

```bash
# Use shared utility for consistent org name and ID generation
PUBLISHER_ID_RESULT=$(bash "${CLAUDE_PLUGIN_ROOT}/scripts/utils/generate-publisher-id.sh" \
  --domain "$domain" \
  --json 2>&1)

if [ $? -eq 0 ]; then
  org_name=$(echo "$PUBLISHER_ID_RESULT" | jq -r '.data.org_name')
  publisher_id=$(echo "$PUBLISHER_ID_RESULT" | jq -r '.data.publisher_id')
else
  # Fallback only if utility fails (Bash 3.x/zsh compatible)
  org_name=$(echo "$domain" | cut -d'.' -f1)
  org_name="$(echo "${org_name:0:1}" | tr '[:lower:]' '[:upper:]')${org_name:1}"
fi
```

**Why this matters:** source-creator.sh generates wikilinks using `generate-publisher-id.sh`. If publisher-generator extracts org_name differently, the publisher ID will mismatch and wikilinks will be broken.

**Extraction Algorithm (in generate-publisher-id.sh):**

1. Strip protocol and www. prefix
2. Extract first component before dot: `researchgate.net` → `researchgate`
3. Capitalize first letter: `researchgate` → `Researchgate`
4. Generate hash from org_name
5. Combine: `publisher-{slug}-{hash}`

## Error Handling

### Missing Domain Field

**Issue:** Source entity lacks `domain` field

**Response:**
```json
{
  "publishers_created": 8,
  "publishers_reused": 2,
  "publishers_failed": 1,
  "by_type": {"individual": 5, "organization": 5},
  "failed_items": [{
    "source": "source-123.md",
    "stage": "creation",
    "reason": "Missing domain field"
  }]
}
```

**Action:** Skip source, continue with others

### Malformed YAML

**Issue:** Source entity has invalid frontmatter

**Response:**
- Skip source
- Log error: "source-789.md: Missing frontmatter"
- Increment `publishers_failed`
- Continue processing

### Script Execution Errors

**Issue:** create-entity.sh fails

**Response:**
- Log error to stderr
- Add to `failed_items` array
- Increment `publishers_failed`
- Continue with next publisher

### Generic Author Names

**Issue:** Author field contains non-specific names

**Detection Logic:**

IMPORTANT: Normalize name to lowercase before matching (case-insensitive matching).
Example: "Staff" → "staff", "ADMINISTRATOR" → "administrator", "Editorial Board" → "editorial board"

Check if normalized name (lowercase) matches any of these patterns:
- `^staff$`
- `^editorial`
- `^editor$`
- `^anonymous$`
- `^various`
- `^contributors?$`
- `^team$`
- `^admin(istrator)?$`  # Matches "admin" or "administrator"
- `^unknown$`
- `^(redaction|editorial board)$`  # Matches either term
- `^news desk$`
- `^wire service$`
- `^the team$`
- `^staff writer$`
- `^news service$`

**Examples:**
- "Staff" → matches `^staff$` → skip
- "Editorial Board" → matches `^editorial` → skip
- "Anonymous" → matches `^anonymous$` → skip
- "Various Authors" → matches `^various` → skip
- "Administrator" → matches `^admin(istrator)?$` → skip
- "Dr. Jane Smith" → no match → create publisher

**Response:**
- Skip individual publisher creation for this author
- Continue processing remaining authors in list
- Fall back to organization publisher only if no valid authors
- Log as informational (not error)

## Wikilink Generation for Sources

**Pattern:** Use `generate-wikilink.sh` utility for workspace-relative wikilinks.

**Implementation:**

For each source in `source_references` array, generate workspace-relative wikilink:

```bash
# Call generate-wikilink.sh via Bash tool
WIKILINK_RESULT=$(bash "${CLAUDE_PLUGIN_ROOT}/scripts/generate-wikilink.sh" \
  --project-path "$PROJECT_PATH" \
  --entity-dir "07-sources" \
  --filename "$source_id" 2>&1)

# Parse result
if [ $? -eq 0 ]; then
  # Success: Extract wikilink from JSON response
  WIKILINK=$(echo "$WIKILINK_RESULT" | jq -r '.data.wikilink')
else
  # Fallback: Use single-project format
  log_conditional WARNING "generate-wikilink.sh failed for $source_id, using fallback"
  WIKILINK="[[07-sources/data/$source_id]]"
fi
```

**Workspace Modes:**

1. **Single-Project Mode** (project_path == workspace_root):
   ```markdown
   - [[07-sources/data/source-climate-bonds-report-abc123]]
   ```

2. **Multi-Project Mode** (project nested in workspace):
   ```markdown
   - [[cogni-research/climate-finance-2024/07-sources/data/source-climate-bonds-report-abc123]]
   ```

**Error Handling:**
- generate-wikilink.sh fails → Log WARNING, use fallback format
- Non-fatal: Publisher creation continues successfully
- Fallback uses single-project format: `[[07-sources/data/{source_id}]]`

**Quality Standards:**
- NEVER construct wikilinks manually (use utility only)
- ALWAYS parse JSON response (don't assume format)
- ALWAYS provide fallback (graceful degradation)
- Log WARNING on fallback, not ERROR

**Tool-Based Execution:**

When implementing via Tool invocations (not bash scripts):
- Use **Bash tool** to call generate-wikilink.sh
- Parse JSON response from stdout
- Extract wikilink from `.data.wikilink` field
- Use **Write/Edit tool** to add wikilink to publisher body
