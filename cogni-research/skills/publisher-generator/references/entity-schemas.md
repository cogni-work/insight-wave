# Publisher Entity Schemas

## Individual Publisher Schema

### Frontmatter Structure

```yaml
---
entity_type: publisher
publisher_type: individual
name: {author_name}
enriched: false
enrichment_status: pending
created_date: {ISO_TIMESTAMP}
source_references:
  - {source_id}
tags: [publisher, publisher-type/individual]
title: "{Publisher name}"
dc:creator: publisher-generator
dc:type: publisher
---
```

## Frontmatter Field Definitions

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `entity_type` | string | Yes | Always "publisher" |
| `publisher_type` | string | Yes | "individual" or "organization" |
| `organization_type` | string | Conditional | Required for organizations, one of 10 categories |
| `name` | string | Yes | Publisher name (author or organization) |
| `enriched` | boolean | Yes | false initially, true after enrichment attempt |
| `enrichment_status` | string | Yes | "pending", "success", "failed", or "minimal" |
| `enrichment_failure_reason` | string | Conditional | Required when enrichment_status is "failed" |
| `created_date` | ISO 8601 | Yes | Timestamp of entity creation |
| `enrichment_date` | ISO 8601 | Conditional | Added during successful enrichment |
| `enrichment_sources` | array | Conditional | URLs used during enrichment (success only) |
| `source_references` | array | Yes | List of source_id values |
| `website` | string | Conditional | Domain for organizations |
| `tags` | array | Yes | Type-specific tags |
| `dc:creator` | string | Yes | Always "publisher-generator" |
| `dc:type` | string | Yes | Always "publisher" |

### Enrichment Status Field Values

| Value | Description | When Used |
|-------|-------------|-----------|
| `pending` | Enrichment not yet attempted | Initial state when entity created |
| `success` | Enrichment succeeded | Web search returned sufficient results, context extracted |
| `failed` | Enrichment failed | Web search failed or returned insufficient results |
| `minimal` | Minimal context provided | Reserved for future use (partial enrichment) |

**Key Trend**: The `enriched` field indicates whether enrichment processing was attempted (true) or not (false). The `enrichment_status` field indicates the outcome of that attempt. This distinction allows:
- Detection of failed enrichments: `enriched: true` + `enrichment_status: failed`
- Detection of successful enrichments: `enriched: true` + `enrichment_status: success`
- Detection of pending enrichments: `enriched: false` + `enrichment_status: pending`
- Prevention of retry loops on failed enrichments (don't re-attempt if `enriched: true`)

## Content Section Requirements

### Individual Publishers

**Unenriched:**
- Section: "Affiliations" (placeholder: "Not yet documented")
- Section: "Sources by This Publisher" (wikilinks to sources)

**Enriched (Success):**
- Section: "Context" (100-400 words)
  - Professional background
  - Areas of expertise
  - Current positions/roles
  - Relevant publications/contributions
- Section: "Sources by This Publisher" (wikilinks to sources)

**Enriched (Failed):**
- Section: "Context" (minimal fallback)
  - "Limited public information available. Publisher details not documented in accessible sources."
- Section: "Sources by This Publisher" (wikilinks to sources)

### Organization Publishers

**Unenriched:**
- Section: "Organization Type" (classification)
- Section: "Related Sources" (wikilinks to sources)

**Enriched (Success):**
- Section: "Context" (100-400 words)
  - Organization mission/mandate
  - Domain expertise
  - Founding/history (brief)
  - Key activities/programs
  - Credibility indicators
- Section: "Related Sources" (wikilinks to sources)

**Enriched (Failed):**
- Section: "Context" (minimal fallback)
  - "Limited public information available. Publisher details not documented in accessible sources."
- Section: "Related Sources" (wikilinks to sources)

## Entity ID Generation

Publishers use semantic slugs with `publisher-` prefix:

```bash
# Generate semantic slug
publisher_id=$(bash "$SCRIPT_GENERATE_SLUG" \
  --title "publisher-$publisher_name" \
  --content-key "publisher-$publisher_name")

# Example outputs:
# publisher-dr-jane-smith-abc123.md
# publisher-climatebonds-def456.md
```

## Deduplication Behavior

When `create-entity.sh` detects existing publisher:

**Response:**
```json
{
  "success": true,
  "entity_id": "publisher-existing-name-abc123",
  "reused": true
}
```

**Entity Updates:**
- Append new source_id to `source_references` array
- Update modification timestamp
- Keep existing frontmatter otherwise

## Validation Requirements

### Pre-Creation Validation

- [ ] Source file has valid YAML frontmatter
- [ ] Domain field present and non-empty
- [ ] Domain field not whitespace-only
- [ ] Publisher name extracted successfully
- [ ] Organization type classified (if applicable)

### Post-Creation Validation

- [ ] Entity file exists in `08-publishers/data/`
- [ ] Frontmatter parseable
- [ ] Required fields present
- [ ] Tags correctly formatted
- [ ] Content sections present
- [ ] Wikilinks valid
- [ ] **⛔ CRITICAL:** `source_references` array is NOT empty (must contain at least one source_id)
- [ ] **⛔ CRITICAL:** `name` field is NOT empty string
- [ ] **⛔ CRITICAL:** `domain` field is NOT empty string (for organizations)

### Post-Enrichment Validation

- [ ] `enriched` field set to true
- [ ] `enrichment_status` field present with valid value
- [ ] If `enrichment_status: "success"`:
  - [ ] `enrichment_date` field present
  - [ ] `enrichment_sources` array present with URLs
  - [ ] Context section has 100-400 words
- [ ] If `enrichment_status: "failed"`:
  - [ ] `enrichment_failure_reason` field present
  - [ ] Context section has minimal fallback text
  - [ ] No `enrichment_date` or `enrichment_sources` fields

### Quality Checks

- [ ] No fabricated publisher names
- [ ] All publishers traceable to source metadata
- [ ] Type detection accurate (individual vs organization)
- [ ] Organization type classification reasonable
- [ ] Deduplication working correctly
- [ ] Enrichment status accurately reflects outcome
