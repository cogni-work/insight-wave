Generate strategic trend entities by synthesizing research data (findings, concepts, megatrends, dimensions) through the lens of research questions. Creates cross-finding trends with inline citations and thematic analysis.

## Purpose

Transform collected research entities into actionable trend documents that answer research questions through **claim-grounded evidence synthesis**. Analyzes all research data (findings, concepts, megatrends, **claims**) to identify patterns, trends, and strategic implications, producing trend entities with numbered citations, **minimum 3 claims per trend**, and complete provenance tracking.

## When to Use

- After deeper-research-1 has populated entity directories (04-findings, 05-domain-concepts, 06-megatrends, **10-claims**)
- When research dimensions (01-research-dimensions) and refined questions (02-refined-questions) exist
- **When 10-claims/data/ contains 30+ verified claims (minimum for proper synthesis)**
- To create cross-finding trends with inline citations, thematic organization, and **claim evidence**
- When synthesizing findings into strategic recommendations and pattern identification
- ALWAYS operates in dimension-scoped mode (one agent per dimension, parallel execution)
- The `--dimension` parameter is MANDATORY - skill will fail without it

**⛔ README Scope:** Each agent writes ONLY `README-{DIMENSION}.md` (e.g., `README-cloud-services.md`).
Do NOT write `11-trends/README.md` - this is generated by orchestrator (`deeper-research-3`) in Phase 8.5.

- **Trend:** Core pattern from evidence (200-250 words)

## ⛔ Claim Integration Requirements

**Every finding/TIPS trend MUST reference minimum 3 claims.**

### Rationale

Claims are verified factual assertions with dual-layer confidence scores. Requiring 3+ claims per trend ensures:

- **Evidence-based synthesis** - Trends grounded in verified data, not opinion
- **Triangulation** - Multiple evidence points increase reliability
- **Verifiable provenance** - Complete audit trail from trend → claim → finding → source

### Selection Criteria

Only use claims meeting quality thresholds:

| Criterion | Threshold | Purpose |
|-----------|-----------|---------|
| `confidence_score` | ≥ 0.75 | High evidence reliability |
| `flagged_for_review` | false | Verified, not pending |
| `claim_quality` | ≥ 0.70 | High linguistic quality |

### Integration Pattern

Each trend must include:

1. **Inline claim quotes** (1+ per major section):

   ```markdown
   "99% of surveyed C-Level managers see supply chain resilience as strategic priority"<sup>[[10-claims/data/claim-supply-chain-99-a1b2|C1]]</sup>.
   ```

2. **Claim Evidence section** (consolidated at end):

   ```markdown
   ## Claim Evidence

   **Key claims supporting this trend:**

   1. "{claim_text}" (confidence: 0.85) [[10-claims/data/claim-id-1|C1]]
   2. "{claim_text}" (confidence: 0.82) [[10-claims/data/claim-id-2|C2]]
   3. "{claim_text}" (confidence: 0.78) [[10-claims/data/claim-id-3|C3]]
   ```

3. **Frontmatter tracking**:

   ```yaml
   claim_refs: [claim-id-1, claim-id-2, claim-id-3]
   ```
