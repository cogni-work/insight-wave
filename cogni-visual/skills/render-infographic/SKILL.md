# Render Infographic — Excalidraw Sketchnote Pipeline

Render an infographic-brief.md (v1.0) into a hand-drawn sketchnote-style Excalidraw scene.
The result looks like a visual notepad: illustrated icons, flowing arrows, organic zones, and
hand-lettered text — NOT a CSS dashboard or HTML page.

Excalidraw's native hand-drawn roughness IS the sketchnote aesthetic. Every element has
slightly imperfect edges, marker-pen feel, and organic visual weight. The infographic composes
150-250 elements across 4-8 visual zones on a single canvas.

## Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `brief_path` | auto-discovered | Path to infographic-brief.md |
| `theme` | from brief frontmatter | Path to theme.md (or omit for pick-theme) |
| `output_path` | `{brief_dir}/infographic.excalidraw` | Output path |
| `language` | from brief frontmatter | `en` or `de` |

## Output

Return single-line JSON:

**Success:**
```json
{"ok":true,"excalidraw_path":"{path}","share_url":"{url}","zones":{N},"total_elements":{count},"style_preset":"{preset}"}
```

---

## Pipeline Overview

```
Phase 1: Parse & Setup           → Extract blocks, resolve theme palette, compute layout grid
Phase 2: Canvas + Page Border    → Clear canvas, draw outer border with title banner
Phase 3: Content Zones           → Draw each zone (rectangle + headline + content elements)
Phase 4: Illustrations           → Compose simple shape-based icons per zone
Phase 5: Flow Connections        → Draw arrows and connectors between zones
Phase 6: CTA + Footer            → CTA bar, source line, metadata
Phase 7: Visual Self-Review      → Screenshot → 6-gate evaluation → fix → repeat (max 3)
Phase 8: Export                   → .excalidraw file + Excalidraw URL + PNG
```

Total: 150-250 elements. Sequential phases, no parallel agents needed.

---

## Phase 1: Parse & Setup

### Step 1.1: Brief Discovery

1. If `brief_path` provided, use directly
2. Otherwise glob for `**/infographic-brief.md` (max 3 levels)
3. If multiple found, present via AskUserQuestion
4. Read brief, validate `type: infographic-brief`, `version: "1.0"`

### Step 1.2: Extract Metadata

From frontmatter: `layout_type`, `style_preset`, `orientation`, `dimensions`, `language`,
`governing_thought`, `customer`, `provider`, `theme_path`.

### Step 1.3: Parse Blocks

Parse the brief body — each `## Block N:` or `## Title Block` section with fenced YAML.
Build a block list: `[{block_type, fields}, ...]`.

### Step 1.4: Resolve Theme Palette

Read theme.md from `theme_path`. Extract the color palette:

```
PALETTE = {
  primary: "#111111",     # Text, borders, headlines
  accent: "#C8E62E",      # Hero numbers, CTA, highlights
  accent_dark: "#8BA31E", # Accent text on light backgrounds
  surface: "#F2F2EE",     # Zone fills
  background: "#FAFAF8",  # Canvas background (or white for whiteboard)
  border: "#E0E0DC",      # Light borders
  text_muted: "#6B7280",  # Sublabels, metadata
  danger: "#D32F2F",      # Negative indicators (comparison left)
  success: "#2E7D32",     # Positive indicators (comparison right)
}
```

### Step 1.5: Compute Layout Grid

**Read reference:** `$CLAUDE_PLUGIN_ROOT/skills/render-infographic/references/01-zone-layout.md`

Compute zone positions based on `layout_type` and block count. The reference file provides
coordinate formulas per layout type.

**Canvas size:** Landscape 1600×1000, Portrait 1000×1600.

Each content block maps to a **zone** — a rectangular area on the canvas. Zones have:
- `x, y` — top-left corner
- `w, h` — width and height
- `block` — the block data
- `zone_style` — border treatment (solid, dashed, none)

Store as `LAYOUT_GRID` array.

## Phase 2: Canvas + Page Border

### Step 2.1: Clear Canvas

```
mcp__excalidraw__clear_canvas()
```

### Step 2.2: Page Background

Create a filled rectangle for the canvas background:

```json
{"type": "rectangle", "x": 0, "y": 0, "width": 1600, "height": 1000,
 "backgroundColor": "{PALETTE.background}", "fillStyle": "solid",
 "strokeColor": "transparent", "roughness": 0}
```

### Step 2.3: Page Border

Slightly rough outer border — the sketchnote "frame":

```json
{"type": "rectangle", "x": 20, "y": 20, "width": 1560, "height": 960,
 "strokeColor": "{PALETTE.primary}", "strokeWidth": 3, "roughness": 1,
 "backgroundColor": "transparent", "fillStyle": "solid"}
```

### Step 2.4: Title Banner

**Read reference:** `$CLAUDE_PLUGIN_ROOT/skills/render-infographic/references/02-element-recipes.md`, section "Title Banner"

From the `title` block: render headline text (large, bold), subline (smaller, muted),
and metadata line. Position centered at top of canvas within the page border.

Elements: 3-5 (headline text, subline text, metadata text, optional underline, optional accent line).

**Snapshot checkpoint** after Phase 2.

---

## Phase 3: Content Zones

**Read reference:** `$CLAUDE_PLUGIN_ROOT/skills/render-infographic/references/02-element-recipes.md`

For each zone in `LAYOUT_GRID`, render the block content as Excalidraw elements.

### Zone Structure (all block types)

Every zone starts with:
1. **Zone border** — rounded rectangle with `roughness: 1`, stroke from palette
2. **Zone headline** — bold text anchored at top-left of zone (if block has a headline)

Then block-type-specific content:

### kpi-card → "Hero Number Zone"

- Large text element for Hero-Number (font-size 48-60, accent_dark color)
- Smaller text for Hero-Label (font-size 16, primary)
- Smallest text for Sublabel (font-size 12, text_muted)
- Optional source text (font-size 10, text_muted, italic)

Elements per zone: 4-6.

### stat-row → "Evidence Strip"

For each stat in the row:
- Number text (font-size 24, accent_dark, bold)
- Label text (font-size 12, text_muted)
- Small circle or shape as icon placeholder

Arranged horizontally within the zone. Elements per zone: 8-15.

### comparison-pair → "Contrast Zone"

- Vertical divider line down the center of the zone
- Left column: label text + bullet items + optional "−" markers in danger color
- Right column: label text + bullet items + optional "+" markers in success color
- Left zone fill with surface color, right zone fill with light accent tint

Elements per zone: 15-25.

### process-strip → "Flow Strip"

For each step:
- Rounded rectangle or circle as step container
- Step label text centered inside
- Arrow element connecting to next step

Arranged horizontally (landscape) or vertically (portrait). Elements per zone: 12-25.

### chart → "Data Sketch"

Charts cannot be rendered as Excalidraw elements directly. Instead, sketch a simplified
visual representation:
- Bar chart → rectangles of varying height with labels
- Doughnut → concentric circles with accent fill
- Line → connected line segments with data points

This is a visual approximation — the goal is to communicate the data trend, not produce
a precise chart. Elements per zone: 10-20.

### text-block → "Annotation Zone"

- Headline text (bold)
- Body text (regular, max 2 lines)
- Optional icon shape

Elements per zone: 3-5.

### icon-grid → "Card Grid"

For each item:
- Small rounded rectangle
- Icon placeholder (circle or shape)
- Label text
- Sublabel text

Arranged in a grid within the zone. Elements per zone: 15-30.

### svg-diagram → "Diagram Zone"

The SVG diagram content is complex — for Excalidraw rendering, compose a simplified version
using rectangles, ellipses, arrows, and text. Follow the data payload in the block to create:
- Hub-spoke: central ellipse + radiating rectangles with arrows
- Process-flow: connected rectangles with arrow chain

Elements per zone: 15-30.

**Batch size:** Send up to 25 elements per `batch_create_elements` call.

**Snapshot checkpoint** after all content zones are rendered.

---

## Phase 4: Illustrations

For each zone that has an `Icon-Prompt` in its block fields, compose a simple shape-based
illustration from Excalidraw primitives (2-5 shapes per illustration).

**Read reference:** `$CLAUDE_PLUGIN_ROOT/skills/render-infographic/references/02-element-recipes.md`, section "Icon Compositions"

These are simple pictograms, not detailed illustrations. Each is 40-60px and composed of
basic shapes:

- "shield" → rectangle with pointed bottom (diamond + rectangle)
- "chart up" → rectangle base + ascending line
- "clock" → circle + two lines (hour/minute hands)
- "person" → circle (head) + rectangle (body)
- "warning" → triangle
- "target" → nested circles
- "brain" → ellipse with curved lines inside
- "arrow up" → arrow element

Position each icon within its zone, typically left of the headline or centered above numbers.

Elements per illustration: 2-5. Total illustration elements: 10-30.

---

## Phase 5: Flow Connections

Draw arrows connecting zones that have a logical flow relationship:
- Process strips: arrows already internal to the zone
- Layout-level connections: the layout type determines inter-zone flow

### Connection Rules by Layout Type

| Layout | Connections |
|--------|------------|
| `timeline-flow` | Left-to-right or top-to-bottom through process strip → supporting zones |
| `comparison` | Arrows from title to both comparison columns; arrows from comparison to evidence |
| `stat-heavy` | Title → hero KPI → chart → evidence; visual hierarchy arrows |
| `hub-spoke` | Central zone radiates arrows to spoke zones |
| `funnel-pyramid` | Top-to-bottom flow arrows between tiers |
| `list-grid` | No inter-zone arrows (grid implies equal weight) |
| `flow-diagram` | Diagram zone contains all connections internally |

Use curved arrows (`type: "arrow"`) with `roughness: 1` for the hand-drawn feel. Arrow color
is `PALETTE.text_muted` or `PALETTE.accent` for primary flow.

Elements: 3-8 arrows.

---

## Phase 6: CTA + Footer

### CTA Block

If the brief has a CTA block:
- Rounded rectangle with accent fill (the "button")
- CTA headline text above
- CTA button text inside the rectangle (bold, contrasting color)

Position below the last content zone. Elements: 3-5.

### Footer

- Horizontal line divider
- Left/center/right text elements
- Source line text (small, italic, text_muted)

Position at bottom of canvas within page border. Elements: 4-6.

**Snapshot checkpoint** after footer.

---

## Phase 7: Visual Self-Review (screenshot → evaluate → fix → repeat)

> **Why:** Excalidraw text auto-sizes and positions differ from what coordinates predict. The
> only reliable way to catch overlaps, clipping, alignment drift, and visual hierarchy
> failures is to look at the actual rendered output. This phase exports a PNG, evaluates it
> against 6 quality gates, fixes failures, and repeats until all gates pass (max 3 iterations).

### Step 7.1: Export Screenshot

```
mcp__excalidraw__export_to_image(format: "png", filePath: "{output_dir}/infographic-review-{iteration}.png")
```

Then read the PNG with the Read tool to visually inspect it.

### Step 7.2: Evaluate Against 6 Quality Gates

Score each gate PASS / WARN / FAIL:

| Gate | What to Check | Pass | Fail |
|------|--------------|------|------|
| **1. Text Readability** | All text legible, no truncation, no overlap between text elements | All text fully visible and readable | Any text overlapping, clipped by zone border, or too small to read |
| **2. Zone Composition** | Each zone has clear internal hierarchy: headline → content → source | Visual hierarchy within every zone reads top-to-bottom naturally | A zone has competing elements or content that overflows its border |
| **3. Visual Balance** | No large empty gaps; no side of the canvas significantly emptier than the other | Weight is distributed across the canvas; all quadrants contain content | One quadrant is visually empty while another is overcrowded |
| **4. Number Prominence** | Hero numbers visually dominate their zones; accent color makes them pop | Hero numbers are the first thing you notice in each KPI zone | Numbers are same visual weight as body text, or lost in surrounding content |
| **5. Flow & Connections** | Arrows guide the eye between zones in a logical reading order | You can trace a natural flow: title → hero KPI → evidence → CTA | Arrows point to empty space, cross over content, or create confusing loops |
| **6. Sketchnote Character** | The infographic looks hand-drawn, not clinical; roughness is visible, Virgil font feels organic | The hand-drawn aesthetic is clearly present; the infographic has visual personality | Everything looks perfectly straight and machine-generated; no sketchnote feel |

### Step 7.3: Fix Failures

For each FAIL or WARN:
- **Text overlap/clipping** → use `update_element` to adjust x/y position or reduce fontSize
- **Zone overflow** → increase zone height/width or shorten text content
- **Empty gaps** → add a small illustration, move a zone, or add a decorative element
- **Number not prominent** → increase fontSize, switch to accent_dark color, add whitespace
- **Arrow misrouted** → delete and recreate with corrected start/end coordinates
- **Too clinical** → increase roughness to 2, add slight dashed borders, verify fontFamily is Virgil (1)

### Step 7.4: Repeat

After fixes, take a new screenshot (Step 7.1) and re-evaluate. Stop when:
- All 6 gates PASS, OR
- 3 iterations completed (diminishing returns beyond this), OR
- Only WARN remains on low-priority gates (5, 6)

Report the review result:
```
Visual review: iteration {N}
Gate 1 (Readability): PASS
Gate 2 (Composition): PASS
Gate 3 (Balance): PASS/WARN
Gate 4 (Numbers): PASS
Gate 5 (Flow): PASS
Gate 6 (Character): PASS
```

**Snapshot checkpoint** after review passes.

## Phase 8: Export

1. Describe scene to verify element count:
   ```
   mcp__excalidraw__describe_scene()
   ```

2. Export the Excalidraw scene:
   ```
   mcp__excalidraw__export_scene()
   ```
   Save to `output_path`.

3. Generate shareable URL:
   ```
   mcp__excalidraw__export_to_excalidraw_url()
   ```

4. Export as PNG for preview:
   ```
   mcp__excalidraw__export_to_image(format: "png")
   ```

5. Report results to user.

## Style Preset → Excalidraw Parameters

**Read reference:** `$CLAUDE_PLUGIN_ROOT/skills/render-infographic/references/02-element-recipes.md`, section "Style Preset Mapping"

| Preset | Roughness | Stroke Width | Zone Border | Zone Fill | Font Feel |
|--------|-----------|-------------|-------------|-----------|-----------|
| `sketchnote` | 2 | 2 | dashed, rounded (20px) | surface color | Hand-drawn (Virgil) |
| `whiteboard` | 1 | 2 | solid, sharp | none (transparent) | Clean (Virgil) |
| `editorial` | 0 | 1 | solid, sharp | none | Clean (Helvetica) |
| `data-viz` | 0 | 1 | solid, subtle | light surface | Clean (monospace for numbers) |
| `corporate` | 0 | 2 | solid, sharp | surface | Clean (Helvetica) |

**Excalidraw fontFamily mapping:**
- Virgil (hand-drawn): fontFamily 1 — for sketchnote, whiteboard
- Helvetica (clean): fontFamily 2 — for editorial, corporate
- Cascadia (monospace): fontFamily 3 — for data-viz numbers
