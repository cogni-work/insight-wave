# Story Worlds

## Purpose

Define the Story World brainstorming method for generating visual scene concepts that transform narratives into integrated illustrated big pictures. Instead of selecting a pre-defined metaphor and overlaying AI images, the skill brainstorms a **Story World** where each station BECOMES an object in the landscape — a broken CNC machine, a sensor-equipped robot arm, a control tower.

## Why This Matters

The difference between a good big picture and a bad one:

- **Bad:** Colored background with white info cards placed on top (cards-on-background)
- **Good:** Cohesive illustrated scene where stations ARE objects in the landscape, guided by inline station numbers and bold headlines

A Story World ensures every visual element serves the narrative. The landscape isn't decoration — it's the story itself, made spatial.

---

## Story World Brainstorming Method

### Three-Phase Process

```
PHASE 1: CONTENT WORLD ANALYSIS
  Read the narrative and extract:
  - Concrete nouns (machines, factories, sensors, platforms)
  - Industry domain (manufacturing, healthcare, logistics)
  - Physical objects mentioned or implied
  - Spatial language (ascending, flowing, connecting, building)
  - Transformation verbs (shift, evolve, replace, modernize)

PHASE 2: CONCEPT GENERATION
  Generate 2-3 Story World concepts. Each includes:
  - world_name: descriptive name (e.g., "Smart Factory Evolution")
  - world_type: literal or lateral
  - world_description: 1-2 sentence scene description
  - station_objects: for EACH station, what it BECOMES in this world
    (object_name + narrative_connection)
  - world_score: narrative_fit (40%) + visual_composability (30%) + brand_fit (30%)

  RULE: At least one literal AND one lateral concept must be generated.

PHASE 3: STATION-OBJECT MAPPING
  For the selected world, finalize each station's landscape object:
  - object_name: what the station becomes (e.g., "Broken CNC Machine")
  - narrative_connection: why this object represents this station's message,
    with concrete visual details for rendering agents
  - scale: hero (1.5x) | standard (1.0x) | supporting (0.8x)

  NOTE: Do NOT specify shape_composition or landscape_composition.
  Rendering agents own the visual interpretation using shape-recipes-v3.md.
```

## Literal vs Lateral Worlds

### Literal World

The narrative's domain IS the scene. No creative leap required — the scene directly depicts the subject matter.

**When to use:** Industry-specific narratives, technical topics, audiences who value directness.

**Examples:**
- Manufacturing narrative → Factory floor scene (machines, conveyors, robots)
- Healthcare narrative → Hospital campus scene (diagnostic devices, care pathways)
- Logistics narrative → Distribution hub scene (containers, trucks, warehouses)

**Advantage:** Immediate recognition, no metaphor to decode.
**Risk:** Can feel literal/prosaic if execution lacks visual dynamism.

### Lateral World

A creative leap where the scene uses a DIFFERENT domain to represent the narrative. The objects are metaphorical but intuitive.

**When to use:** Abstract topics, transformation stories, audiences who value creativity.

**Examples:**
- Digital transformation → Rocket launch scene (each station = a rocket component being assembled)
- Market expansion → Expedition scene (base camp → summit, equipment at each station)
- Platform strategy → Cityscape scene (buildings = capabilities, roads = integrations)

**Advantage:** More visually engaging, creates surprise and memorability.
**Risk:** Audience must "get" the metaphor; too abstract = confusion.

## 6 Classic Worlds as Inspiration

These are starting points for brainstorming, not a fixed selection menu. Each demonstrates how landscape elements become station objects.

### 1. Mountain / Ascent World

**Scene concept:** Landscape rises from valley to summit. Stations are landmarks along the climb.

**How objects integrate:**
| Narrative concept | Becomes in scene |
|-------------------|-----------------|
| Current problem | Broken bridge at valley floor |
| Growing urgency | Rockslide blocking the trail |
| Key evidence | Weather station with readings |
| Solution | Cable car / engineered path |
| Proof | Mountain hut with view |
| Call to action | Summit flag / beacon |

**Spatial flow:** Bottom-left to top-right (ascending)
**Best for:** Challenge → transformation → achievement arcs

### 2. River / Flow World

**Scene concept:** Water flows from source to sea. Stations are features along the waterway.

**How objects integrate:**
| Narrative concept | Becomes in scene |
|-------------------|-----------------|
| Starting state | Narrow mountain spring |
| Challenge | Rapids / dam |
| Turning point | Confluence of tributaries |
| Solution | Engineered canal / lock |
| Growth | Widening river with ports |
| Destination | Harbor / delta |

**Spatial flow:** Left to right (flowing)
**Best for:** Evolution, lifecycle, and process arcs

### 3. Road / Highway World

**Scene concept:** Road stretches through diverse terrain. Stations are landmarks and intersections.

**How objects integrate:**
| Narrative concept | Becomes in scene |
|-------------------|-----------------|
| Origin | Small town / starting gate |
| Milestone | Signpost / gas station |
| Decision point | Highway interchange |
| Capability | Bridge over gorge |
| Result | City landmark visible |
| Future | Horizon line / sunset |

**Spatial flow:** Left to right (linear or gently curving)
**Best for:** Roadmaps, milestones, implementation plans

### 4. Archipelago / Islands World

**Scene concept:** Islands in a sea, connected by bridges or boats. Stations are island features.

**How objects integrate:**
| Narrative concept | Becomes in scene |
|-------------------|-----------------|
| Core theme | Central large island |
| Topic A | Island with lighthouse |
| Topic B | Island with harbor |
| Topic C | Island with observatory |
| Connection | Bridge / shipping lane |
| Conclusion | Beacon visible from all islands |

**Spatial flow:** Hub-and-spoke or stepping stones
**Best for:** Portfolio overviews, parallel themes, ecosystem views

### 5. Garden / Growth World

**Scene concept:** Garden transitions through seasons. Stations are garden features at different growth stages.

**How objects integrate:**
| Narrative concept | Becomes in scene |
|-------------------|-----------------|
| Problem | Bare soil / winter patch |
| Early action | Seedlings / greenhouse |
| Cultivation | Irrigation system / trellis |
| Bloom | Full garden in flower |
| Harvest | Fruit trees / harvest basket |
| Future | Greenhouse / new planting |

**Spatial flow:** Left to right (seasonal progression)
**Best for:** Growth, nurturing, development arcs

### 6. Cityscape / Urban World

**Scene concept:** City panorama with districts and infrastructure. Stations are buildings and structures.

**How objects integrate:**
| Narrative concept | Becomes in scene |
|-------------------|-----------------|
| Legacy state | Old building / crumbling facade |
| Disruption | Construction crane / demolition |
| New capability | Modern office tower |
| Integration | Bridge / transit hub |
| Monitoring | Control center / observatory |
| Vision | Green district / skyline |

**Spatial flow:** Left to right (district traversal) or radial
**Best for:** Technology, modernization, digital transformation

## World Scoring

When generating 2-3 Story World concepts, score each on three dimensions:

```
world_score = narrative_fit * 0.40 + visual_composability * 0.30 + brand_fit * 0.30

narrative_fit (0.0 - 1.0):
  How naturally do the narrative's key concepts map to objects in this world?
  - 1.0: Every station has an obvious, intuitive object
  - 0.7: Most stations map well, 1-2 are a stretch
  - 0.4: Requires significant creative leaps
  - 0.1: Forced / unnatural mapping

visual_composability (0.0 - 1.0):
  How well can this world be built from Excalidraw primitives?
  - 1.0: All objects are simple geometric compositions (rectangles, circles, lines)
  - 0.7: Most objects are composable, 1-2 need complex shapes
  - 0.4: Many objects require detailed illustration beyond primitives
  - 0.1: The world demands photorealistic or highly organic shapes

brand_fit (0.0 - 1.0):
  How well does this world match the customer's brand and context?
  - 1.0: Perfect match (tech company + cityscape, manufacturer + factory)
  - 0.7: Good fit, natural association
  - 0.4: Neutral — neither enhances nor conflicts
  - 0.1: Mismatch (serious financial services + cartoon garden)
```

### Scoring Rules

1. **Literal worlds** typically score higher on `narrative_fit` but may score lower on visual surprise
2. **Lateral worlds** can score higher on memorability but must still achieve >= 0.5 on `narrative_fit`
3. **Industrial/technical narratives** strongly favor literal worlds unless the audience values creativity
4. **Minimum threshold:** Discard any world scoring below 0.50 total

## Visual Style Mapping

When `visual_style` is `auto`, select based on narrative context:

| Customer Context | Recommended Style | Roughness |
|-----------------|-------------------|-----------|
| Enterprise / corporate | `flat-illustration` | 0 |
| Creative / agency | `sketch` | 2 |
| Technical / engineering | `flat-illustration` | 0 |
| Startup / innovation | `sketch` | 1 |
| Government / public sector | `flat-illustration` | 0 |
| Education / non-profit | `sketch` | 1 |

When `visual_style` is explicitly set, use it regardless of context.

### Roughness as Style (Excalidraw)

| Roughness | Feel | When to Use |
|-----------|------|-------------|
| 0 | Corporate, clean, precise | Enterprise, government, technical |
| 1 | Warm, approachable, hand-drawn | Startups, education, creative |
| 2 | Sketch-like, creative, loose | Workshops, brainstorming, informal |
